# AudioLinkCore システム構造分析

`AudioLinkCore` は、プロジェクト全体の「耳」の役割を果たすコンポーネントです。入力された音声信号を周波数帯域や特性ごとに分離（ステム分離）し、各シーンが扱いやすい 0.0 〜 1.0 の数値データへと変換する役割を担っています。

## 全体フロー
1.  **入力 (`in1`)**: 入力デバイスまたはファイルからの生音。
2.  **スペクトル解析 (`demucs_spectrum`)**: 音声信号を多重の周波数帯域、または Demucs（AI分離）的なロジックで複数のストリームへ分割。
3.  **パラレル解析 (各 parse コンポーネント)**: 各ストリームから特定の楽器成分を抽出。
4.  **統合・出力 (`out1` / `out1_export`)**: 全てのデータを1つの CHOP にまとめ、外部コンポーネントが参照できるように提供。

---

## 主要モジュールと出力チャンネル

### 1. Kick (キック)
*   **ノード**: `parse_kick`
*   **役割**: 低域の瞬間的なアタック（トランジェント）を抽出。
*   **主な出力**: `kick`

### 2. Snare / Hihat / Clap (高域パーカッション)
*   **ノード**: `parse_snare`
*   **役割**: スネアの胴鳴りや、ハイハットの鋭い高域を分離。
*   **主な出力**: `Snare`, `Hihat`, `Clap`

### 3. Bass (ベース)
*   **ノード**: `parse_bass`
*   **役割**: ベースの持続的な低音エネルギーや、サイドチェイン的な動きを抽出。
*   **主な出力**: `uBassEnergy`, `usubass`, `usostenido`

### 4. Vocal (ボーカル)
*   **ノード**: `parse_vocal`
*   **役割**: 中音域の調波成分（歌声など）を抽出。
*   **主な出力**: ボーカルのエネルギー値

### 5. Melody (メロディ)
*   **ノード**: `parse_melody`
*   **役割**: メロディラインやリード楽器の動きを追跡。

---

## 独自の調整ポイント
*   **Callbacks (`..._callbacks`)**: 各解析モジュールの挙動（感度や減衰速度など）は、それぞれのコンポーネントに付随する Python DAT (Callbacks) で定義されている可能性があります。
*   **Monitoring**: 下部の `Math` -> `Audio Device Out` は、現在どの音が解析対象になっているかを耳で確認するためのモニター経路です。

## SCENE_5 での参照
現在 SCENE_5 の `audio_data` (Select CHOP) は、この `AudioLinkCore/out1` を参照して、`uBassEnergy` と `Hihat` を取得しています。
